# Recommendations

Recommendations are a durable opportunity backlog, not an executable task list.

A recommendation may come from the user, an assessment, a drift review, an incident, an audit, or a completed change.

## Many-to-many links

- A broad recommendation may be delivered through several changes.
- A coherent change may satisfy several recommendations.
- Store lists in `Related recommendations` and `Converted changes`.
- Each linked change proposal stores `Source recommendations`.
- Do not mark a recommendation `Completed` until its intended outcome is fully addressed or explicitly closed.

Use `prompts/03-capture-or-triage-recommendations.md` for triage and `prompts/04-create-approved-change.md` for conversion.
