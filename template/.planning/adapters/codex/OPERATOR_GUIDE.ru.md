# Codex: краткая памятка оператора

- `$planning-dialogue`: критический разбор без правок кода.
- `$planning-plan`: определение или планирование change.
- `$planning-execute`: исполнение уже готового и прямо разрешённого change.
- `$planning-audit`: аудит, readiness, drift или closure.
- `$planning-checkpoint`: записать состояние перед `/compact` или `/new`.

Skill явно выбирается для текущего запроса. После новой беседы вызовите нужный skill снова и попросите прочитать `AGENTS.md` и активное состояние.

Команды интерфейса могут различаться по версии Codex. Агент готовит checkpoint, но `/compact`, `/new`, выбор модели и разрешения выполняет пользователь.
