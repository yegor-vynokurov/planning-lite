# Codex CLI: памятка оператора

Этот файл предназначен для пользователя. Агент не должен загружать его в каждый запрос.

Команды могут зависеть от версии Codex и выбранной поверхности. Введите `/`, чтобы увидеть доступные команды в вашей установке.

## Контекст и расход токенов

| Команда | Что делает | Когда использовать |
|---|---|---|
| `/status` | Показывает модель, режим доступа, использование контекста и лимиты | После планирования и после крупной партии задач |
| `/statusline` | Настраивает постоянную строку со статистикой контекста, токенов, модели и Git | Один раз при настройке CLI |
| `/usage` | Показывает дневное, недельное или суммарное использование аккаунта | Для контроля общего расхода, не только текущего окна |
| `/compact` | Сжимает раннюю историю текущей задачи в резюме | После записанного checkpoint, когда продолжается тот же этап |
| `/new` | Начинает новую беседу без старой истории | Новый change, новая фаза или слишком загрязнённая история |
| `/mention <path>` | Явно добавляет конкретный файл | Вместо прикрепления каталога или повторного чтения репозитория |
| `/ide-context` | Включает или выключает автоматический IDE-контекст | Отключить, если открытые вкладки раздувают запрос |
| `/mcp` | Показывает подключённые MCP-серверы и инструменты | Проверить, не загружены ли ненужные инструменты |
| `/memories` | Управляет использованием и созданием Memories, когда функция доступна | Отключить для изолированной задачи или проверить источник лишнего контекста |

## Режим работы

| Команда | Что делает | Когда использовать |
|---|---|---|
| `/plan` | Переводит текущую задачу в plan mode | Перед многошаговым планированием, пока код писать нельзя |
| `/reasoning` | Выбирает глубину рассуждения текущей задачи | `low` для мелких правок, `medium` для исполнения, `high` для архитектуры и риск-аудита |
| `/goal <цель>` | Прикрепляет постоянную цель к активной задаче | Для длинного change; подробности держать в файле, а не в goal |
| `/skills` | Позволяет выбрать repo skill | Для `$planning-plan`, `$planning-execute` и других режимов каркаса |
| `/model` | Меняет модель текущей задачи | Более сильная модель для архитектуры, более экономная для механических шагов |

## Разделение разговоров и проверка

| Команда | Что делает | Когда использовать |
|---|---|---|
| `/side` | Временный боковой разговор, не загрязняющий основную задачу | Короткий вопрос или проверка риска во время длинной работы |
| `/fork` | Копирует текущую задачу в отдельную ветку разговора | Сравнение альтернативных архитектур или экспериментов |
| `/review` | Запускает review незакоммиченных изменений или сравнение с веткой | Перед закрытием change или после рискованной партии |
| `/diff` | Показывает текущий Git diff | Использовать выборочно; для системной проверки предпочесть `$planning-git-review` |
| `/resume` | Возвращает сохранённую беседу | Когда нужен именно старый transcript, а не чистая сессия по checkpoint |

## Правило перед `/compact` или `/new`

Сначала скажите агенту одну из коротких команд:

```text
Выполни чекпоинт перед /compact.
Выполни чекпоинт перед /new.
```

Или вызовите `$planning-checkpoint`. Агент обновит durable state и соберёт компактную Git-сводку без полного raw diff. После ответа агента используйте `/compact` или `/new` сами. Если review обязателен, новая сессия должна начать с `$planning-git-review` до продолжения реализации или закрытия change.

## Экономный Git review

Для проверки изменений используйте `$planning-git-review` или команду `Проведи Git review изменений с последнего чекпоинта`. Workflow сначала смотрит статус, статистику, имена файлов и bounded commit range, а затем открывает только рискованные patches. Регулярные атомарные коммиты позволяют проверять небольшой диапазон вместо всего branch diff.

## Рекомендуемая глубина `/reasoning`

- `minimal` или `low`: форматирование, переименование, простая локальная правка.
- `medium`: исполнение утверждённого плана и обычная отладка.
- `high`: архитектурный выбор, миграция, безопасность, неоднозначный brownfield-анализ.
- `xhigh`: только для действительно сложной задачи и если выбранная модель его поддерживает.

Высокая глубина не нужна постоянно. Она повышает расход и может замедлить механическую работу.
