# Refresh project state after out-of-band changes

Mode: Audit.

Use when code, documentation, data, or architecture changed outside the normal workflow, including manual user edits.

## Procedure

1. Identify what changed and why from diff, commits, user explanation, tests, and artifacts.
2. Determine whether each edit is:
   - harmless alignment;
   - quick fix that belongs in the ledger;
   - plan amendment;
   - scope amendment;
   - separate recommendation or change;
   - unauthorized risky work requiring recovery.
3. Update architecture, current state, glossary, rules, and completion evidence where reality changed.
4. Create a dated assessment when impact is material.
5. Update recommendations and many-to-many links.
6. Do not falsify approved change history. Use retrospective records honestly.
7. Run drift sync when thresholds or immediate triggers apply.
8. Summarize remaining decisions and next permitted action.

Do not rewrite code merely to match an obsolete plan.
