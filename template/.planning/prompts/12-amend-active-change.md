# Amend an active change

Mode: Planning.

## Procedure

1. State what changed in reality and why the current artifacts no longer fit.
2. Classify the amendment:
   - Plan detail;
   - Scope;
   - Acceptance criteria;
   - Emergency safety.
3. Add an entry to `amendments.md` without rewriting original approval history.
4. For a plan-detail amendment:
   - update plan, tasks, progress, and context;
   - rerun targeted audit;
   - continue only if no new critical blocker exists.
5. For scope or acceptance amendments:
   - draft updates to proposal/specification;
   - show changed requirements, non-goals, risk, migration, compatibility, and tests;
   - mark affected execution blocked;
   - obtain explicit user approval before applying and resuming.
6. Update linked recommendations when coverage changes.

## Output

State amendment type, rationale, affected artifacts, approval requirement, and next permitted action.
