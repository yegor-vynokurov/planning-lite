# Capture or triage recommendations

Mode: Dialogue / critic or Planning.

## Procedure

1. Use the index first. Load only named or closely related recommendations.
2. Capture durable ideas from the user, assessment, audit, incident, drift review, or completed change.
3. Separate evidence, interpretation, recommendation, expected outcome, risks, and alternatives.
4. Add `Related recommendations` as a list when items overlap, depend on one another, conflict, or may be bundled.
5. Propose priority, impact, effort, and confidence, but do not mark `Accepted` without the user’s decision.
6. When accepted direction is ready for conversion, decide with the user whether it should become:
   - one bounded change;
   - several sequenced changes;
   - one change combining several recommendations.
7. Update `Converted changes` in every source recommendation only after the linked change exists.
8. Do not place executable tasks in recommendations.

## Status rules

- `Accepted`: direction approved, no implementation implied.
- `Converted`: at least one linked change exists; outcome may still be partial.
- `Completed`: intended outcome fully delivered or explicitly closed.

## Output

Summarize new or changed IDs, links, decisions, unresolved conflicts, and next permitted action.
