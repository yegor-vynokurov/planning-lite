## Rules

- `Plan`: implementation detail, sequence, file path, or verification improvement without changing approved outcome. Record before continuing.
- `Scope` or `Acceptance`: changes requirements, non-goals, users, compatibility, data, security, or acceptance criteria. Stop affected execution until the user approves.
- `Emergency safety`: immediate non-destructive action needed to prevent loss or exposure. Record what was done and request review promptly.
- Never rewrite old approval history to make an amendment look original.
