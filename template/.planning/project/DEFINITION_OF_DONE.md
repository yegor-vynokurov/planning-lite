# Definition of Done

Apply only relevant categories and record `Not applicable` explicitly.

## Scope and behavior

- Mandatory acceptance criteria pass.
- Delivered behavior matches approved scope and non-goals.

## Code and architecture

- Repository conventions and architectural boundaries are respected.
- No unexplained dead paths, hidden scope, or accidental public-contract changes remain.

## Verification

- Required tests, checks, and observable evidence are recorded.
- Failures and skipped checks are explicit.

## Data, migration, recovery, and repeated execution

- Applicable migration, rollback, recovery, idempotence, and compatibility behavior are verified.

## Documentation and operations

- User, developer, deployment, monitoring, and support documentation are updated where affected.

## Project bookkeeping

- Tasks, progress, context, amendments, review, recommendations, decisions, project state, and ACTIVE agree.
