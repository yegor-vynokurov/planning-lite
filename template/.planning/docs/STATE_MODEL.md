# State model

The active pointer is small and records lifecycle stage, stage status, next gate, authorization, and context path. Detailed mutable state stays with the active change. Historical evidence remains in progress and Git. Project documents describe durable facts and domain language. Recommendations describe future opportunities. Decisions preserve rationale. Optional skill-usage logs are observational only.

See `.planning/control/CHANGE_LIFECYCLE.md` for transitions and `.planning/control/STATE_OWNERSHIP.md` for the normative ownership table.
